# .roorules - BorderComfy 專案工作規則

## 語言與溝通
- 所有溝通須使用繁體中文（zh-TW）
- 文件、程式碼註解、輸出皆使用繁體中文

## 工作流程
- **分析優先**：在進行任何編碼之前，先分析問題並提出報告
- **根本處理**：正確地、從根本地處理任務，而不是用補丁（patch）方式臨時應對
- **完整規劃**：先建立 TODO list，確認每個步驟的邏輯順序，再進入實作階段

## 任務執行原則
1. **理解問題**：深入了解需求和現有架構
2. **規劃架構**：設計清晰的技術架構和資料流程
3. **逐步執行**：按照邏輯順序執行每個步驟
4. **驗證結果**：每個階段完成後驗證結果再繼續

## 專案特定規則
- 這是 Wix 遷移專案，需要識別並處理所有 Wix 特定的依賴
- 所有外部資源（圖片、字體、JS）必須下載並內嵌到本地
- 保持網站功能和設計一致性
- 確保遷移後網站可脫離 Wix 基礎設施獨立運行
